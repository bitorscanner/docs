# Getting Started with Bitor

## Prerequisites

Bitor requires Ansible and Terraform for automation and infrastructure management. Here's how to install them:

### macOS

```bash
# Install Homebrew if you haven't already
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Ansible and Terraform
brew install ansible
brew install terraform
```

### Linux (Ubuntu/Debian)

```bash
# Install Ansible
sudo apt update
sudo apt install -y ansible

# Install Terraform
sudo apt-get update && sudo apt-get install -y gnupg software-properties-common
wget -O- https://apt.releases.hashicorp.com/gpg | gpg --dearmor | sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### Linux (RHEL/CentOS)

```bash
# Install Ansible
sudo dnf install -y ansible

# Install Terraform
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://rpm.releases.hashicorp.com/RHEL/hashicorp.repo
sudo yum -y install terraform
```

## Quick Installation

### macOS

For Apple Silicon (M1/M2) Macs:
```bash
# Download the macOS ARM64 version
wget https://github.com/bitorscanner/bitor/releases/download/v0.5.0/bitor_0.5.0_Darwin_arm64.tar.gz

# Extract the archive
tar -xzf bitor_0.5.0_Darwin_arm64.tar.gz
```

For Intel Macs:
```bash
# Download the macOS Intel/x86_64 version
wget https://github.com/bitorscanner/bitor/releases/download/v0.5.0/bitor_0.5.0_Darwin_x86_64.tar.gz

# Extract the archive
tar -xzf bitor_0.5.0_Darwin_x86_64.tar.gz
```

After extracting, continue with:
```bash
# Make it executable
chmod +x bitor

# Set your encryption key (must be 32 characters)
export API_ENCRYPTION_KEY=12345678901234567890123456789012

# Start Bitor
./bitor serve

# Access the web interface at
http://localhost:8090
```

### Windows Support

:::caution[Windows Limitations]
Windows is not natively supported due to dependencies on Ansible and Terraform. These tools are designed for Unix-like systems and have limited Windows support. We recommend using one of these alternatives:
- Docker installation (recommended for Windows users)
- Windows Subsystem for Linux (WSL2) with Ubuntu
- Running in a Linux virtual machine

If using WSL2, you can follow the Ubuntu installation instructions above within your WSL2 environment.
:::

## Docker Installation

Docker provides the easiest way to run Bitor across all platforms.

```bash
# Pull the Bitor image
docker pull ghcr.io/bitorscanner/bitor:latest

# Run Bitor container
docker run -d \
  -p 8090:8090 \
  -e API_ENCRYPTION_KEY=12345678901234567890123456789012 \
  ghcr.io/bitorscanner/bitor:latest

# Access the web interface at
http://localhost:8090
```

### Docker Compose

For a more complete setup, you can use Docker Compose:

```yaml
# docker-compose.yml
version: '3.8'
services:
  bitor:
    image: ghcr.io/bitorscanner/bitor:latest
    ports:
      - "8090:8090"
    environment:
      - API_ENCRYPTION_KEY=12345678901234567890123456789012
    volumes:
      - bitor_data:/data
    restart: unless-stopped

volumes:
  bitor_data:
```

Run with:
```bash
docker-compose up -d
```

## Troubleshooting

If you encounter any issues:

1. Check the logs with `./bitor logs`
2. Ensure all prerequisites are installed
3. Verify your encryption key is exactly 32 characters
4. Join our Discord for community support
